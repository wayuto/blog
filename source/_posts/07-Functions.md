提到`函数`这一概念，对于没有学习过计算机语言的人来说，可能会想到数学中的函数：一次函数，二次函数……，对于给定的`自变量`，具有一个唯一的`函数值`，在函数式编程中，同样存在这一概念，即：`无副作用函数`，但在过程式编程中，`函数`通常是有副作用的，也就是同样的输入，每次可能会得到不同的输出。在`Alum`中，一个函数是这样定义的：
```alum
fun F(Pn: PTn, ...): RT expression
```
其中`F`是函数的名字，就像`sin`，`cos`这样的，`Pn`是参数名，`PTn`是参数类型，`RT`是函数返回的类型。一个函数可以有多个参数，但只能有一个返回值。`expression`即函数体，需要注意的是，`Alum`的`{` `}`与其中包含的多个语句实际上组成一个单独的表达式(如果你想要了解代码块，可以回顾[05-Block](05-Block.md))，也就是说，`Alum`并不强制你必须用一个`{` `}`的代码块构成函数体，也可以是一个单独的`return`语句（在`Alum`中，必须显式的`return`，如果是`void`也要`return nil`）。
例如：
```alum
fun succ(n: int): int return n + 1
```
这样是完全可以的，甚至如果你想的话，也可以这么写：
```alum
fun fib(n: int): int return {
    if n < 2 {
        n
    } else {
        fib(n - 1) + fib(n - 2)
    }
}
```
这与
```alum
fun fib(n: int): int {
    if n > 2 {
        return n
    } else {
        return fib(n - 1) + fib(n - 2)
    }
}
```
的运行效果是一样的，但还是推荐后者，这样具有更好的可读性
